1️⃣ Add Keyboard Controls (Desktop/Web)

Currently Velocity uses swipe gestures for movement.

Add keyboard controls so the game can be tested on laptops.

Supported keys:

ArrowUp → dodge up  
ArrowDown → dodge down  
ArrowLeft → dodge left  
ArrowRight → dodge right  

Also support:

W → up  
A → left  
S → down  
D → right

Implementation details:

• Use a keydown event listener
• Only enable keyboard controls on web or desktop
• Keyboard input must call the SAME dodge handler used by swipe gestures
• Do not change existing mobile gesture logic

Optional UI hint for web:

Show a small hint text at the bottom:

"Use arrow keys to dodge"

Only display this when Platform.OS === "web".

2️⃣ Add Edge Warning Indicators

Before an obstacle spawns, show a directional warning flash.

Example:

Obstacle coming from LEFT:
• flash a red glow along the left edge
• pulse once for ~200ms

Obstacle from TOP:
• flash glow along top edge

Implementation:

Create a reusable component:

components/EdgeWarning.tsx

Props:
direction: "left" | "right" | "top" | "bottom"
visible: boolean

Visual style:

• glowing neon edge bar
• color based on obstacle color
• quick pulse animation using Reanimated
• fade in → fade out

This warning should appear about 300–500ms before the obstacle moves.

3️⃣ Add Player Orb Trail Effect

components/OrbTrail.tsx

Use:
