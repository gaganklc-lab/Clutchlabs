==================================================
CORE GAME DESIGN TO PRESERVE
==================================================

Velocity is a swipe-to-dodge survival game.

Player orb starts in the center.
Obstacles come from one of four directions:
- top
- bottom
- left
- right

Correct dodge rule:
- obstacle from top -> dodge down
- obstacle from bottom -> dodge up
- obstacle from left -> dodge right
- obstacle from right -> dodge left

Keep this rule exactly.

==================================================
2) DESKTOP / WEB FALLBACK CONTROLS
==================================================

Add keyboard support for laptop/web testing.

Support:
- ArrowUp = dodge up
- ArrowDown = dodge down
- ArrowLeft = dodge left
- ArrowRight = dodge right
- W = up
- A = left
- S = down
- D = right

Requirements:
- keyboard input must use the SAME dodge resolution logic as swipe
- enable only on web/desktop
- do not break mobile controls
- show a small hint on web:
  "Use arrow keys or WASD to dodge"

If possible, also support mouse drag swipe on web, but keyboard is the priority.

==================================================
3) TABLET SUPPORT
==================================================

Make the Velocity game look good on iPad / tablet.

Requirements:
- center content with a max content width
- do not stretch game UI edge-to-edge on tablet
- keep the play area large but intentional
- preserve safe margins
- increase spacing slightly on tablet
- ensure HUD, hint text, power-up bar, and play area do not feel crowded

Use useWindowDimensions() and tablet detection:
- isTablet = width >= 768

Recommended:
- contentMaxWidth around 560–640
- contentHorizontalPadding larger on tablet

4) VISUAL DODGE FEEDBACK
==================================================

The orb must visibly dodge.

Currently the game logic works, but the orb does not clearly move when the player succeeds.

Add:
- orb dash animation on successful dodge
- dash in the actual dodge direction
- return smoothly to center after a short delay
- preserve failure shake animation

Examples:
- obstacle from left + correct right dodge -> orb visibly dashes right
- obstacle from top + correct down dodge -> orb visibly dashes down

Requirements:
- keep dash fast and satisfying
- do not make the orb stay permanently off-center
- reset orb cleanly between obstacles
